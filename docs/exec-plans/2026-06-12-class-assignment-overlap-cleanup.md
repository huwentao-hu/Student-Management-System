# 执行计划：分班历史重叠技术债清理

日期：2026-06-12  
状态：已完成

## 目标

确保同一学生的任意分班历史日期区间都不能重叠，避免课程表错误合并多个班级课程。

## 执行步骤

1. 检查真实 PostgreSQL 是否已有重叠数据。
2. 服务层对学生分班写入加锁，并校验完整历史。
3. PostgreSQL 增加日期范围排他约束。
4. 补充重叠历史回归测试。
5. 验证 API 和直接 SQL 均无法产生重叠。
6. 更新相关文档并关闭技术债。

## 验收条件

- 入班、转班和离班不能产生重叠历史。
- 同一学生的并发分班写入按学生串行处理。
- PostgreSQL 直接写入重叠记录时由数据库拒绝。
- 课程表不会因重叠分班历史错误合并班级。

## 完成内容

- 检查真实 PostgreSQL，确认迁移前没有重叠分班历史。
- 服务层分班写入改为先锁定学生记录，再校验完整历史。
- 新增 Flyway Java V7 迁移。
- PostgreSQL 启用 `btree_gist` 并增加日期范围排他约束。
- 新增关闭历史后回填重叠日期的回归测试。
- 新增同一学生并发分班写入测试。

## 验证结果

- `backend\mvnw.cmd test`：通过，28 个测试，0 失败。
- H2 上 Flyway V1 至 V7：执行成功。
- PostgreSQL 上 Flyway V7：执行成功。
- API 重叠分班请求：返回 `400 Bad Request`。
- PostgreSQL 直接 SQL 重叠插入：被 `ex_class_assignments_no_overlap` 排他约束拒绝。
