# 执行计划：成绩与考勤并发首次写入清理

日期：2026-06-12  
状态：已完成

## 目标

让同一业务记录的并发首次成绩和考勤写入稳定完成，并保持单条最终记录。

## 执行步骤

1. 明确并发写入的串行化规则。
2. 在成绩写入前锁定学生记录。
3. 在考勤写入前锁定学生记录。
4. 补充成绩和考勤并发写入测试。
5. 使用真实 PostgreSQL 验证。
6. 更新所有相关文档和技术债状态。

## 验收条件

- 同一学生同一开课安排的并发成绩写入均成功。
- 同一学生同一课堂的并发考勤写入均成功。
- 每组并发写入最终只保留一条记录。
- 自动化测试和真实 PostgreSQL 验证通过。

## 完成内容

- 成绩录入或修改前使用学生行悲观写锁。
- 考勤录入或修改前使用学生行悲观写锁。
- 同一学生并发首次写入由创建和后续更新组成，不再出现偶发唯一约束错误。
- 保留成绩和考勤数据库唯一约束作为最终兜底。
- 清理 `GradeService` 未使用导入。
- 未新增数据库表或 Flyway 迁移。

## 验证结果

- `backend\mvnw.cmd test`：通过，41 个测试，0 失败。
- H2 上 Flyway V1 至 V9：执行成功。
- 真实 PostgreSQL 并发成绩首次写入：返回 `201 Created` 和 `200 OK`，最终 1 条记录。
- 真实 PostgreSQL 并发考勤首次写入：返回 `201 Created` 和 `200 OK`，最终 1 条记录。
