# 执行计划：后端交付准备

日期：2026-06-12  
状态：已完成

## 目标

为前端联调和基本运行检查补齐公共健康检查、主动退出和可配置 CORS。

## 执行步骤

1. 实现数据库健康检查接口。
2. 实现当前令牌主动退出。
3. 配置允许来源和浏览器预检请求。
4. 补充健康检查、令牌撤销和 CORS 集成测试。
5. 使用真实 PostgreSQL 验证。
6. 更新所有相关文档。

## 验收条件

- 未登录可查询健康检查并确认数据库正常。
- 已登录用户退出后当前令牌立即失效。
- 退出不撤销同一账号的其他令牌。
- 允许来源的浏览器预检请求返回正确 CORS 响应头。
- 未允许来源不会获得 CORS 允许响应头。
- 自动化测试和真实 PostgreSQL 验证通过。

## 完成内容

- 新增公共数据库健康检查接口。
- 新增当前登录令牌主动退出接口。
- 新增可通过环境变量配置的 CORS 允许来源。
- 浏览器预检请求无需认证，受保护业务请求仍需 Bearer Token。
- 未新增数据库表或 Flyway 迁移。

## 验证结果

- `backend\mvnw.cmd test`：通过，39 个测试，0 失败。
