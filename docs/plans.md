## 已确认规则

- 系统只服务一所学校，不需要多学校数据隔离。
- 系统角色为管理员、教师、学生。
- 管理员管理所有数据。
- 教师查询学生，后续管理成绩与考勤。
- 学生登录后仅查看自己的信息。
- 学号由系统自动生成，创建后不可修改。
- 系统存在不同班级，每个班级记录班主任。
- 学生不必须归属班级，转班和离班保留历史记录。
- 课程目录与授课安排分离。
- 课程编号由系统生成，课程名称全校唯一。
- 开课安排关联课程、班级、授课教师、学年和学期。
- 同一班级在同一学年学期不能重复安排同一课程。
- 学生课程表根据指定学期内的分班历史计算，学生只能查询本人课程表。
- 成绩采用百分制，同一学生对同一开课安排最多一条成绩。
- 管理员管理全部成绩，授课教师管理自己课程的成绩，学生查看本人。
- 成绩统计及格线固定为 60 分，教学班统计可由管理员和该授课教师查看。
- 学生学期成绩汇总采用课程学分加权平均，管理员可查看全部，学生只查看本人。
- 考勤按课堂场次记录，状态为出勤、迟到、请假或缺勤。
- 管理员管理全部考勤，授课教师管理自己的课堂考勤，学生查看本人。
- 考勤统计以已登记记录为分母，出勤与迟到计入出勤率，未登记学生不自动视为缺勤。
- 管理员可查看全部考勤统计，授课教师查看自己教学班统计，学生查看本人学期汇总。
- 健康检查无需登录并检查数据库连通性。
- 用户可主动退出并仅撤销当前登录令牌。
- 浏览器跨域允许来源通过环境变量配置，预检请求无需登录。
- 成绩与考勤写入使用学生行锁，同一学生的并发首次写入串行执行。
- 过期登录令牌默认每小时自动清理，管理员也可手动清理。
