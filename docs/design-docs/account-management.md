# 账号管理设计

状态：已实现  
最后更新：2026-06-12

## 目标

允许管理员创建教师账号，并为已有学生创建学生账号，使教师和学生能够使用登录接口进入系统。

## API

`POST /api/accounts`

要求携带管理员 Bearer Token。成功返回 `201 Created`。

## 请求规则

| 字段 | 规则 |
| --- | --- |
| `username` | 必填，3 至 64 字符，全局唯一 |
| `password` | 必填，8 至 128 字符，只保存 BCrypt 哈希 |
| `role` | 只允许 `TEACHER` 或 `STUDENT` |
| `studentId` | 学生账号必填；教师账号禁止填写 |

## 业务规则

- 只有管理员可以创建账号。
- 不允许通过该接口创建管理员账号。
- 学生账号必须绑定已有学生。
- 每名学生最多绑定一个账号。
- 创建成功后账号立即启用并可以登录。

## 错误响应

- 未登录：`401 Unauthorized`
- 非管理员调用：`403 Forbidden`
- 请求角色或学生关联不合法：`400 Bad Request`
- 用户名重复或学生已经绑定账号：`409 Conflict`
- 学生不存在：`404 Not Found`

## 数据库影响

本功能复用 `user_accounts` 表，未修改数据库结构。
