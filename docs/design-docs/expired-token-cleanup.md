# 过期登录令牌清理设计

状态：已实现  
最后更新：2026-06-12

## 目标

定期删除已经过期且未主动退出的登录令牌，避免 `auth_tokens` 表持续积累无效数据。

## 清理规则

- 删除 `expiresAt <= 当前时间` 的令牌。
- 默认每小时自动清理一次。
- 自动清理周期可通过 `AUTH_TOKEN_CLEANUP_INTERVAL_MS` 配置。
- 应用启动后首次自动清理延迟可通过 `AUTH_TOKEN_CLEANUP_INITIAL_DELAY_MS` 配置。
- 清理操作不会删除仍然有效的登录令牌。

## 管理员接口

`POST /api/auth/tokens/cleanup`

- 仅管理员可调用。
- 使用当前时间立即执行一次清理。
- 返回本次删除的过期令牌数量。

## 数据实现

使用数据库条件删除直接清理过期记录，不先加载令牌实体。无需新增数据库表或 Flyway 迁移。
