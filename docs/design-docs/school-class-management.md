# 班级管理设计

状态：已实现  
最后更新：2026-06-13

## 目标

管理学校中的不同班级，并为每个班级记录班主任教师。

## 数据模型

`school_classes` 表包含：

| 字段 | 规则 |
| --- | --- |
| `name` | 必填，最长 100 字符 |
| `entryYear` | 必填，1900 至 2200 |
| `homeroomTeacherId` | 必填，必须指向已启用的教师账号 |

同一入学年份内班级名称唯一。同一教师可以担任多个班级的班主任。

## API

### 创建班级

`POST /api/classes`

仅管理员可调用。

```json
{
  "name": "软件工程1班",
  "entryYear": 2026,
  "homeroomTeacherId": 2
}
```

### 查询班级

- `GET /api/classes/{id}`
- `GET /api/classes?keyword=软件&entryYear=2026&page=0&size=20`
- `GET /api/classes/{id}/students`：查询当前花名册

管理员和教师可查询；学生暂不开放班级管理查询。

## 错误响应

- 班级不存在：`404 Not Found`
- 同一入学年份班级名称重复：`409 Conflict`
- 班主任账号不存在、不是教师或已禁用：`400 Bad Request`
- 非管理员创建：`403 Forbidden`

## 学生分班规则

- 学生不必须始终归属班级。
- 转班和离班保留完整历史记录。
- 学生入学年份是否必须与班级入学年份一致。

## 前端实现

- 管理员和教师可查询班级列表并查看详情。
- 列表支持班级名称搜索和入学年份筛选。
- 仅管理员显示新增班级入口。
- 新增班级通过启用教师下拉框选择班主任。
- 班级详情展示当前花名册，并可进入学生详情。
- 花名册只包含当前分班记录，历史离班学生不在名单中。
