# 成绩与考勤并发写入设计

状态：已实现  
最后更新：2026-06-12

## 目标

清理同一学生同一开课安排成绩、同一学生同一课堂考勤的并发首次写入冲突，避免数据库唯一约束异常返回通用服务错误。

## 处理规则

- 成绩和考勤接口继续采用录入或修改的 `PUT` 语义。
- 写入前使用数据库悲观写锁锁定学生记录。
- 同一学生的并发成绩或考勤写入串行执行。
- 首个请求创建记录，后续请求读取已有记录并更新。
- 最终只保留一条成绩或考勤记录，不返回偶发唯一约束冲突。
- 不同学生的写入不相互阻塞。

## 数据库保障

- `grades` 继续使用学生与开课安排唯一约束兜底。
- `attendance_records` 继续使用课堂场次与学生唯一约束兜底。
- 无需新增数据库表或迁移。
