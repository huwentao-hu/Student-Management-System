# 学生分班历史设计

状态：已实现  
最后更新：2026-06-13

## 目标

允许学生暂时没有班级，并完整记录学生入班、转班和离班历史。

## 数据模型

`student_class_assignments` 每条记录包含学生、班级、开始日期和可选结束日期。

- `endDate` 为空表示当前班级。
- 学生同一时间只能有一个当前班级。
- 同一学生的任意历史日期区间不能重叠。
- 结束日期不得早于开始日期。
- 历史记录不会因转班或离班删除。

## API

### 入班或转班

`POST /api/students/{studentId}/class-assignments`

```json
{
  "classId": 1,
  "effectiveDate": "2026-09-01"
}
```

如果学生已有当前班级，系统会把旧记录结束日期设置为新班级生效日期的前一天。

### 离班

`POST /api/students/{studentId}/class-assignments/leave`

```json
{
  "effectiveDate": "2027-07-01"
}
```

离班后学生没有当前班级，旧班级记录仍保留。

### 查询历史

`GET /api/students/{studentId}/class-assignments`

历史按开始日期倒序返回。

分班历史也用于计算学生在指定学年学期的课程表。

## 权限

- 管理员：入班、转班、离班和查询历史。
- 教师：查询所有学生的分班历史。
- 学生：只查询自己的分班历史。

## 日期规则

新的转班或离班生效日期必须晚于当前分班记录的开始日期。新增分班会检查完整历史，生效日期不能落入或早于任何尚未结束的历史区间。

分班写入会锁定学生记录，使同一学生的并发操作串行执行。PostgreSQL 额外使用日期范围排他约束，阻止直接 SQL 产生重叠历史。

## 待确认

学生入学年份是否必须与班级入学年份一致。

## 前端实现

- 学生详情展示当前班级和按开始日期倒序排列的完整分班历史。
- 管理员可选择目标班级和生效日期执行入班或转班，也可按生效日期离班。
- 教师只能查看分班历史，不显示写入操作。
