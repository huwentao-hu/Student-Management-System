# 课程管理设计

状态：已实现  
最后更新：2026-06-13

## 目标

建立全校统一的课程目录，供后续开课安排、成绩和考勤模块引用。

课程目录只描述课程本身。授课教师、上课班级和学期属于后续开课安排，不直接记录在课程表中。

## 数据模型

`courses` 表包含：

| 字段 | 规则 |
| --- | --- |
| `courseCode` | 系统生成，格式为 `C` + 8 位数据库主键序号，创建后不可修改 |
| `name` | 必填，最长 100 字符 |
| `credits` | 必填，范围 0.5 至 30.0，最多一位小数 |
| `status` | 必填，`ACTIVE` 或 `INACTIVE`，默认 `ACTIVE` |

课程名称在全校课程目录中唯一。

## API

### 创建课程

`POST /api/courses`

仅管理员可调用。

```json
{
  "name": "数据库原理",
  "credits": 3.5,
  "status": "ACTIVE"
}
```

### 查询课程

- `GET /api/courses/{id}`
- `GET /api/courses?keyword=数据库&status=ACTIVE&page=0&size=20`

管理员和教师可查询；学生暂不开放课程目录查询。

## 错误响应

- 课程不存在：`404 Not Found`
- 课程名称重复：`409 Conflict`
- 参数不符合规则：`400 Bad Request`
- 非管理员创建：`403 Forbidden`

## 后续扩展

开课安排已单独实现，用于关联课程、授课教师、班级和学期。后续可增加课程修改、停用和学生课程表。

## 前端实现

- 管理员和教师可查询课程目录、搜索课程名称或编号并按状态筛选。
- 管理员可在课程管理页新增课程，课程编号由系统生成。
- 课程详情展示基本资料和该课程已有开课安排。
