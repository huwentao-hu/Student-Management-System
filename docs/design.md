# 系统设计规范

最后更新：2026-06-12

## API 规范

- API 根路径使用 `/api`。
- 资源名称使用复数，例如 `/api/students`。
- 新增成功返回 `201 Created`，并提供 `Location` 响应头。
- 查询成功返回 `200 OK`。
- 参数校验失败返回 `400 Bad Request`。
- 资源不存在返回 `404 Not Found`。
- 唯一性冲突返回 `409 Conflict`。
- JSON 字段使用 `camelCase`。
- 分页参数从 `page=0` 开始，默认 `size=20`，最大 `size=100`。
- 分页响应统一包含 `content`、`page`、`size`、`totalElements`、`totalPages`、`first`、`last`。
- 登录失败统一返回 `401 Unauthorized`，不区分用户名不存在或密码错误。
- 未携带有效登录令牌返回 `401 Unauthorized`。
- 已登录但权限不足返回 `403 Forbidden`。
- 主动退出成功返回 `204 No Content`，当前令牌立即失效。
- `GET /api/health` 无需登录并检查数据库连通性；数据库不可用时返回 `503 Service Unavailable`。

## 数据库规范

- 表名和列名使用 `snake_case`。
- 业务主表使用 `BIGINT` 自增主键。
- 关键业务编号使用唯一约束，例如学生学号。
- 学号由系统使用入学年份与数据库主键生成，创建后不可修改。
- 所有结构变化通过 `backend/src/main/resources/db/migration/` 下的新迁移完成。
- 已执行的迁移文件不可直接修改。
- 主表记录 `created_at` 和 `updated_at`。
- 系统只服务一所学校，业务表不增加 `school_id`。
- 密码只保存 BCrypt 哈希。
- 原始登录令牌只返回给客户端，数据库保存 SHA-256 哈希。

## 后端代码规范

- 按业务模块组织代码，不按技术层建立全局大目录。
- Controller 只负责 HTTP 协议与参数校验。
- Service 负责业务规则和事务。
- Repository 负责数据访问。
- 请求和响应使用独立 DTO，不直接暴露实体。
- 统一异常响应由 `common/GlobalExceptionHandler` 处理。

## 认证与权限规范

- 固定角色为 `ADMIN`、`TEACHER`、`STUDENT`。
- 每个账号只有一个角色。
- 学生账号必须关联学生档案；管理员和教师账号不关联学生档案。
- 默认登录令牌有效期为 8 小时。
- 首个管理员仅在设置 `BOOTSTRAP_ADMIN_PASSWORD` 时自动创建。
- 受保护接口使用 `Authorization: Bearer <token>`。
- `POST /api/auth/logout` 只撤销当前 Bearer Token，不影响同账号其他登录令牌。
- 过期登录令牌默认每小时自动清理，周期与首次延迟可通过环境变量配置。
- `POST /api/auth/tokens/cleanup` 仅管理员可调用，并返回删除的过期令牌数量。
- CORS 允许来源通过 `CORS_ALLOWED_ORIGINS` 配置，浏览器预检请求无需 Bearer Token。
- 学生接口权限：管理员可读写，教师只读全部，学生只读自己的单条档案。
- 账号创建接口仅管理员可调用。
- 账号创建接口只允许创建教师和学生账号，不允许创建管理员账号。
- 学生账号必须绑定已有学生，且一名学生只能拥有一个账号。
- 班级创建仅管理员可操作，管理员和教师可查询。
- 班级必须关联已启用的教师账号作为班主任。
- 同一入学年份内班级名称唯一。
- 学生可没有当前班级。
- 入班、转班和离班必须保留带起止日期的历史记录。
- 同一时间学生只能有一个当前班级。
- 同一学生的所有分班历史日期区间不得重叠。
- 分班写入必须锁定学生记录，PostgreSQL 同时使用日期范围排他约束。
- 课程创建仅管理员可操作，管理员和教师可查询。
- 课程编号由系统生成且创建后不可修改。
- 课程名称全校唯一，课程目录不直接关联教师、班级和学期。
- 开课安排创建仅管理员可操作，管理员和教师可查询。
- 开课安排只允许关联启用课程和已启用教师账号。
- 同一班级在同一学年学期不能重复安排同一课程。
- 学生只能查询自己的课程表，管理员和教师可代查任意学生。
- 学生课程表根据学期日期范围内的分班历史和班级开课安排实时计算。
- 成绩采用 0.0 至 100.0 的百分制，同一学生同一开课安排最多一条。
- 成绩录入前必须校验学生在该学期内归属过开课班级。
- 成绩写入必须锁定学生记录，同一学生的并发录入或修改串行执行。
- 管理员可管理全部成绩，授课教师只管理自己负责开课安排的成绩，学生只查询本人。
- 成绩统计及格线为 `60.0`，平均分、学分加权平均分和及格率保留两位小数。
- 管理员可查询全部成绩统计，授课教师只能查询自己教学班统计，学生只能查询本人学期汇总。
- 学生学期成绩汇总按课程学分计算加权平均分。
- 成绩统计使用数据库聚合查询，不加载完整成绩实体集合。
- 同一开课安排同一天首版只允许一个课堂考勤场次。
- 学生考勤状态包括 `PRESENT`、`LATE`、`EXCUSED` 和 `ABSENT`。
- 登记考勤时学生必须在课堂日期当天归属开课班级。
- 考勤写入必须锁定学生记录，同一学生的并发登记或修改串行执行。
- 管理员可管理全部考勤，授课教师只管理自己的课堂考勤，学生只查询本人。
- 考勤统计以已登记记录为分母，`PRESENT` 和 `LATE` 计入出勤，出勤率保留两位小数。
- 管理员可查询全部考勤统计，授课教师只能查询自己教学班统计，学生只能查询本人学期汇总。
- 未登记考勤的学生首版不自动视为缺勤。
- 考勤统计使用数据库聚合查询，不加载完整考勤实体集合。

## 测试规范

- 每个小功能至少覆盖成功路径和一个关键失败路径。
- 自动化测试使用 H2 PostgreSQL 兼容模式快速执行。
- 数据库迁移或数据库相关功能完成后，至少使用真实 PostgreSQL 验证一次。

## 前端规范

- 使用 React、TypeScript、Vite 和 React Router。
- 前端 API 地址只通过 `VITE_API_BASE_URL` 配置，不硬编码生产域名。
- 角色导航用于改善体验，所有权限仍由后端强制校验。
- 页面必须支持桌面端和移动端，提供加载、空数据和错误状态。
- 每次前端修改至少执行 `npm run lint` 和 `npm run build`。
- Cloudflare Pages 构建目录为 `frontend/dist`，单页路由使用 `_redirects` 回退。
